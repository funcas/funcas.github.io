盛最多水的容器题解记录

发表于 2020-04-18 分类于 算法与数据结构 Valine：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

题目

给你 n 个非负整数 a1，a2，...，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂直线 i 的两个端点分别为 (i, ai) 和 (i, 0)。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

说明：你不能倾斜容器，且 n 的值至少为 2。

示例：

输入：[1,8,6,2,5,4,8,3,7] 输出：49

解法一：暴力穷举

这应该是短时间内最容易被想出来的方法了，类似冒泡排序，从头到尾两两做组合，并计算组合出来的面积大小，保留最大值，遍历结束即为结果。

计算面积的方法，从题目中不难观察出来，底为两柱间距，高以短板为准，所以得出以下求面积的计算逻辑：

min(height[i], height[j]) * (j - i)

上代码(go语言描述）

逻辑简单易懂就不多做解释，时间复杂度嘛：O($ n^{2} $），遇上数据量大的话，超时是必然了，虽然亲测LeetCode的测试点并不会超时，大概在500ms左右的样子。

解法二 双指针法

考虑一下如果我们大脑在思考这个问题求解的时候是怎么处理的，直觉上会直接淘汰掉那些距离很近、高度差别过大的组合。这也是人脑的优势所在，不需要计算出精确数据就能做相对精确的比较，剩下的差别不太大，肉眼不容易分辨的再做计算，挑出最大的。于是，我们是不是考虑让程序来完成这种相应的操作呢。我们把思考这个过程做下抽像，其实我们在这个过程中寻找的需要符合这样的情况：两柱间距离尽量大，两柱高度相关不太大的，这样围出来的面积就是接近于最大的存在。

双指针法的思路大概是这样的，定义头尾两个指针，比较两个指针所指向的柱高，柱短的那个指针如果是左指针就向前移动一位，如果是右指针就向后移动一位，直到左指针位置大于右指针的位置，结束循环。 初始状态如下图，假定：绿色表头指针，红色表尾指针：

那么此时头尾指针间的面积为： > maxArea = mix(1, 7) * (8 - 0) = 1 * 8 = 8

拉下来可以有两种移动方法，移动柱高的指针或移动柱矮的指针，我们假设移动柱高的那个指针，即图中的尾指针（红色）

这时候的面积为：

maxArea = mix(1, 3) * (7 - 0) = 1 * 7 = 7

发现什么规律了没？在移动的过程中，底是以1为步长依次递减，而高则是以短板为准，也就是说，柱高决定了整块容器面积，所以移动柱高的那个指针时，柱高度并不参与到计算过程中，所以面积会越来越小，而永远不会超过第一次的状态下的这个面积了，于是这部分的组合可以直接过滤掉。所以我们在移动的过程中，应该要移动的是柱矮的那个指针，即现在的头指针（绿色） 第一次移动后如下图：
